# Система рекомендаций загрузки оборудования

Система помогает планировать и распределять задания аддитивного производства по оборудованию и операторам. Интерфейс синхронизирован с API и отражает связи сущностей, заложенные в базе данных.

## Возможности

- Планирование заданий с учётом приоритетов, длительности, времени подготовки и завершения.
- Управление оборудованием, его типами и характеристиками.
- Управление операторами, компетенциями и закреплёнными устройствами.
- Визуализация загрузки оборудования и графиков работы.
- Администрирование пользователей и доступов.

## Стек

- Backend: Go
- База данных: PostgreSQL
- Frontend: HTML/CSS/JS (статическая панель)

## Краткая структура данных

Ключевые связи БД (см. `internal/storage/createDB.sql`):

- **Workspace** связывает пользователей, оборудование и задания.
- **Характеристики оборудования** → **типы оборудования** → **устройства**.
- **Операторы** связаны с **компетенциями** по типам оборудования и закреплёнными устройствами.
- **Задания** привязаны к оборудованию, оператору, приоритету и типу задания.
- **User tasks** фиксируют персональные поручения внутри заданий.

## Быстрый старт

### Через Docker Compose

Из корня проекта:

```bash
docker compose up --build
```

Сервис API будет доступен по адресу `http://localhost:8080`, фронтенд — на `http://localhost:8080/`.

### Локальный запуск

1. Поднимите PostgreSQL и создайте базу с параметрами из `.env`.
2. Проверьте переменные окружения (пример — `.env`).
3. Запустите backend:

```bash
cd recsys-backend
go run ./cmd/app
```

## Интерфейс

- Страница **Главная** — оперативная статистика и загрузка оборудования.
- **Задания** — диаграмма планирования и статусы выполнения.
- **Оборудование** — карточки устройств с состоянием и участием в рекомендациях.
- **Операторы** — компетенции, закреплённые устройства и ближайшие задания.
- **Справочники** — управление характеристиками, типами и справочными данными.
